# Kitty Reschedule Alerts Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Reliably flag reschedule requests, notify Swati on WhatsApp, and prevent agent payload/template-shape errors.

**Architecture:** Normalize allowlisted payload aliases at the authenticated tools boundary, construct scheduling template parameters in the sender, and dispatch an internal-only idempotent Swati alert through the existing signed WhatsApp sender. Keep `hermes_messages` as the durable delivery ledger and expose a separate admin case listing action.

**Tech Stack:** Next.js 16 route handlers, TypeScript, Supabase Postgres, Node test runner, Python Hermes plugins, Meta WhatsApp Cloud API, Vercel, Fly.io.

## Global Constraints

- Camel-case is canonical; compatibility aliases are allowlisted and shallow.
- Student/tutor authorization and case membership remain fail-closed.
- Agents never provide parameters for `admin_reschedule_alert`.
- Meta `accepted` is not described as `delivered`.
- Financial templates are outside this change and remain backend-generated by the settlement module.
